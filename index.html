Simple version-enumeration system with git

If you like how git "does all the work for you" concerning merging, you might like how this simple script will "do all the work for you" concerning managing version numbers.

NOTICE: This project was started before github was around, but now has moved to GITHUB... where it BELONGS! Long live github!

How it works

This script is an **easy** way to tag & branch releases on a remote git repositiory that follows the mainline-versioning method described in the Perforce High-level Best Practices in Software Configuration Management document. Specifically, with a single command any user with the ability to perform a "git-push" to the blessed "release server" can make a new release (by either a branch or tag). Many advanced git users may find this script totally unnecessary.

It runs with bash & git. Please don't laugh at my shell-scripting abilities, instead send improvements.

Example of what happens by simply running "./release.sh" when at certain spots of a local repo that tracks the version server. Automatically generating the appropriate branches, tags and release numbers.

One can see how this might encourage single version branches (as below from the aforementioned Perforce document).
